Metaprogramming

Metaprogramming is the writing of computer programs with the ability to treat programs as their data. It means that a program could be designed to read, generate, analyse and/or transform other programs, and even modify itself while running.

Elixir

  • compile-time metaprogramming
  • the compiler starts with an AST that resembles this code. This AST is then expanded to produce the final code
  • Elixir AST fragment: {operation, context, arguments}
  • Expansion phase
  • Tools: quote, unquote, __using__, __before_compile__

sum_expr = quote do: a + b

bind_expr = quote do
    a = 1
    b = 2
  end

final_expr = quote do
  unquote(bind_expr)
  unquote(sum_expr)
end

# $> iex
  • We produced a new AST fragment that conbines both expressions

iex1> defmodule Tracer do
       #...
      end

iex2> require Tracer

iex3> quoted = quote do Tracer.trace(1+2) end

iex4> expanded = Macro.expand(quoted, __ENV__)

iex5> Macro.to_string(expanded) |> IO.puts
    

defmodule MyModule do
  require Tracer

  #...
  def some_fun(...) do
    Tracer.trace(...)
  end
end
  

defmodule MyModule do
  require Tracer
  ...
  def some_fun(...) do
    result = 1 + 2
    Tracer.print("1 + 2", result)
    result
  end
end
The point of all this is to demonstrate that your macro call is really expanded to something else. This is how macros work. Though we only tried it from the shell, the same things happen when we’re building our projects with mix or elixirc.
