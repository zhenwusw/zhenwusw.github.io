Why We Should Learn FP

Sep 15th, 2014

  • It tell computer what to do instead of how to do something

    simple example FP prefer recursion instead of while , for loops pattern matching instead of a lot of if else test for differet conditions

    maximum' :: (Ord a) => [a] -> a maximum' [] = error "maximum of empty list" maximum' [x] = x maximum' (x:xs) | x > maxTail = x | otherwise = maxTail where maxTail = maximum' xs

  • separate type declaration and function definition

  • pattern matching vs imperative language’s if/else or other polymorphisms strategies of class inheritance system
  • elegant function guard and where bindings
  • recursion
