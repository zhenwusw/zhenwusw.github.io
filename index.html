Metaprogramming

  • Writing code that writes code
  • Dynamic metaprogramming vs Static metaprogramming
  • Elixir/Clojure Macro systems

Ruby

Dynamic programming language

  • Ruby has it’s own VM call YARV: tokenizing, parsing, compiling, executing
  • Ruby doesn’t do type checking ** code optimization are done at converting AST into YARV instructions

static language: Type checking, register allocation, code generation, and code optimization are typically done at compile time

Pure OO

  • numbers and other primitive types are not objects in some other languages
  • Everything is object: that’s an important rule
  • instance variable + methods to all types of objects
  • operators are syntactic sugar for methods: e.g. + (smiliar)

Flexible

  • Allow users to freely alter its parts at runtime
  • Single inheritance + Mixin
  • Functional support via Closures: block, lambda, proc which can be attached to any method, heavily used in ruby’s Enumerable module
  • dynamic query information + alert definitions

Not making a deal between you and compiler, defining class in Ruby is actually running code

Adding / removing methods: it can deprecate many boilerplate code, such as getter and setter. In some other OO languages, Class acts like a contract/template, it allows to spawn many instances as it describes, but doesn’t allow itself to be modified. So you need to specify every single method in this template definition, we call it boilerplate methods. Ruby totally avoid this via define_methods, method_missing …

Ruby object model

  • regular objects: instance variables, class pointer
  • class objects: class instance variables, class pointer, superclass pointer

class inheritance + mixin MyClass.ancestors => ...

singleton methods vs instance methods vs class methods singleton class vs metaclass

All metaclasses are singleton classes, but not all singleton classes are metaclasses. Ruby automatically creates a metaclass for every class you create and uses it to hold class methods that you might declare later.

  4. instance_methods
  5. instance_eval + class_eval
  6. Hooks: inherited, included, extend_object, method_added, method_removed, method_undefined

  • Expansion phase: compiler use quoted expression to generate final bytecode
  • Tool: quote + unquote

How can Elixir code run before it is produced?

It can’t. To call a macro, the container module must already be compiled, that’s why require or import is needed.

When we require a module, we instruct the Elixir to hold the compilation of the current module until the required module is compiled and loaded into the compiler run-time (the Erlang VM instance where compiler is running)

Jan 29th, 2014

Homoiconic

The property of code being data and data being code is called homoiconicity. The representation of a program is also a data structure in a primitive type of the language.

  • function that takes data as arguments and returns data
  • data can b code representation
  • it’s called at compile time
  • code what it’s invoked will be replaced
  • powerful, performant way to build DSLs
  • Achieve many of the same effects as Metaprogramming but at compile time: Allow us to be lazy about evaluation function calls: arguments are always resolved before they are passed to them

Abstract Syntax Trees(AST)

AST is a tree representation of the syntactical structure of source code. It is referred to as abstract since it does not exactly represent every detail that appears in the real syntax. Since the representation is now focused on the machine and not humans, there may be more optimal modeling that lose some of the details of the source.

  • quote provides functionality to turn code into AST
  • unquote injecte variable values into the AST
  • unquote_splicing turns a list* of arguments into comma delimited arguments
    in the code
  • Macro Escaping:
  • Eval quoted: add a runtime aspect to our compile time macro though with some limited scope
  • hygiene: it’s about scope of your variables
  • unquoting outside of macros: The bind_quoted will disable unquote and thats exactly what we want, it’s clear that unquote is not part of the macro.

Binding and unquote fragments

  • Unquote fragments provide an easy way to generate functions on the fly

  • This is because the macro is expecting its arguments to be a keyword list at compilation time.

    ┃ kv = [foo: 1, bar: 2] ┃ defkv kv

The code will fail because we are passing the representation of variable kv

In practice, we want to avoid doing work at compilation time as much as possible.
